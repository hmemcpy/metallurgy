# IntelliJ 2026.1 as the minimum platform version

Metallurgy targets IntelliJ 2026.1 (`since-build="261"`). Earlier IntelliJ versions are not supported.

Three things force the floor upward:

1. The bundled Scala plugin's recent compile-server messaging uses the EEL path APIs (`UpdateCompilerGeneratedStateListener.scala:32-43`), which are 2025.x+.
2. The declarative inlay hint API (`codeInsight.declarativeInlayProvider`) we depend on for Phase 4 has stabilised only on recent platform versions.
3. JBR 25 (which 2026.x ships — `JBR-25.0.3+9-329.124-jcef`) is the runtime users will actually have; `build.sbt` targets `--release 17` bytecode so the plugin loads on any JBR ≥ 17, but the supported runtime is the JBR 25 that ships with IntelliJ 2026.1.

The trade-off is the obvious one: users on 2024.x and 2025.x are excluded. The audience for "Scala 3 IDE support on top of `pc`, on top of BETASTy" is overwhelmingly on recent IntelliJ versions, so we accept the smaller reach in exchange for a cleaner API surface and no fallback code paths.

We track the absolute latest stable (today, 2026.1), not the EAP that the in-development bundled plugin targets (2026.2). Users on the prior stable get one version of grace.
