# Scala 3.5.0 as the minimum supported Scala version

Metallurgy's floor is Scala 3.5.0, the first version with BETASTy (best-effort TASTy, enabled by `-Ybest-effort`). Earlier Scala 3 versions are explicitly unsupported.

BETASTy is the core primitive that lets `pc` see useful types and symbols from upstream modules that don't currently compile — the IDE's normal state. Without it, `pc` can do best-effort typing *within a file* but degrades badly *across modules*. Picking 3.5.0 as the floor lets us design around BETASTy being always available, simplifying the architecture (no graceful-degradation matrix, no per-version feature gating).

The trade-off is real: Scala 3.3 LTS users on patches older than the BETASTy backport are excluded. We accept that cost because (a) Scala 3 users typically follow the latest stable releases, (b) the backport to LTS is in progress per the Scala team, and (c) supporting 3.3.x without BETASTy would require maintaining a parallel "best-effort within file only" code path.
